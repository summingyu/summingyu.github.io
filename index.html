发表于 2016-11-23  |  2016-11-23   |   分类于 linux   |  

写在前面

现在服务程序一般都使用双机备份，防止出现单点故障，而备份不可能每天手动去备份，所以要使用备份程序进行同步。

实施过程

打通2.52与2.51之间的ssh免密码登陆(root用户)

  1. 在各个服务器使用命令ssh-keygen -t rsa -P '' -f /root/.ssh/id_rsa一次生成或者运行ssh-keygen -t rsa然后敲击三次回车生成密钥对
  2. 如果在各个服务器上的/root/.ssh/目录下没有authorized_keys文件，则生成：touch authorized_keys
  3. 修改权限chmod 600 authorized_keys
  4. 将其他服务器上生成的id_rsa.pub追加到本地authorized_keys文件中
    1. 使用scp命令将远程服务器上的id_rsa.pub下载到本地服务器

      scp -p root@192.168.2.52:/root/.ssh/id_rsa.pub /root/.ssh/id_rsa.pub_192.168.2.52
